The first 90 days.

Okay, I'm about as entry-level as possible for this field, why should you consider me? Here's an example of my first 90 days at a company.

While the basis of the job is standard IT work, I've set myself apart by focusing on security posture improvement. As an ISC2 member I'm familiar with the standards and frameworks that exist in the field, and have no interest in paying lipservice to security posture.

I was inspired by an interview with Tyler Farrar, CISO of Exabeam Video here regarding his first 90 days as CISO. I sat there thinking, "I want to make a difference, I could do this."

Within the first 90 days of my employment I set out to configure CIS controls (version 8) and get everything compliant to various frameworks. Eventually, drastically changing the security posture for the better. Leveraging existing tools that had been bought and paid for, yet gone unused, I set to work. First I implemented MFA org-wide, ensuring client were protected and that we had OTP configured for all of our administrative accounts.

Second, I configured geographic drop rules for all Sophos appliances and mail-transfer agents, which provided a degree of seperation from automated attacks. Notably, one particular Barracuda device was under a constant barrage of external threats from various Yandex IPs.

Okay, what's next? Log fatigue. Endless IAM alerts and billing renewal notifications all centralized to a single support email. Leveraging SaaS alerts I configured an array of rules using their 'Respond' module, manual review reserved for Conditional access violations and suspicious behaviour.

At this point, it seemed prudent to configure BitLocker. Using a combination of Microsoft Endpoint Manager for AAD and powershell for on-premise customers, I rolled out BitLocker to several hundred endpoints over the course of a few weeks with no issues. This granted a modicum of data protection at rest locally.

I felt accomplished despite, one final glaring red 'F' taunting me, well I use high-contrast mode, so I assume it was red. Cylance by BlackBerry was our EDR designed to replace Windows Defender. Every month we get an itemized report from a SOC team that had been hired to monitor it, leaving the configuration and tuning of the EDR to our technicians. Great. Script control, memory protection etc. So essentially, EDR isn't configured, but deployed organization wide. I set out to configure a single directory that could still execute script processes, then meticulously responded to every violation in the backlog. After a few months, we were sitting at a comfortable "B" grade, "A" was unfortunately reserved for 100% compliance, which seemed impossible given our existing standard operating procedures and habits.
